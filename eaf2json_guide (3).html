EAF → JSON 変換ガイド

exercise_ant.eaf を例に、utterance / sentence / morph / w_id / extw_id / POS / word_pos の対応関係を図解

1

Utterance と Sentence の対応

EAF の text 層の各タイムスロット区間が 1つの utterance（発話）になる。 sentence_id は boundary の EOS を検出するたびにインクリメントされ、 複数の utterance にまたがって連番で振られる。1つの utterance 内に複数の文が含まれることもある。

EAF text 層 → JSON utterance / sentence_id

utterance 1
tittja-i kodomo otokonoko ga sentence_id = 1
utterance 2
ano jookan=ni sun-de=masi-te, EAC → sentence_id = 1
Sentence 0
Sentence 1
ルール: 句読点（. ,）は morphs から除去され、 gloss の EOS → boundary EOS（文末）、 EOA → boundary EAC（節末）として直前の形態素に付与される。 sentence_id は EOS が現れたときだけインクリメントし、EAC ではインクリメントしない。
2

形態素 / extw_id / w_id の対応

text 層の表記に含まれる区切り文字（スペース・-・=）が ID の境界を決定する。 extw_id はスペースで区切られた「拡張語」の単位、 w_id はスペースと = で区切られた「語」の単位を表す。

⎵

extw_id と w_id の両方をインクリメント。独立した語の境界。
-
ハイフン（接辞境界）
extw_id も w_id もそのまま。同一語内の形態素境界。
=
イコール（接語境界）
extw_id はそのまま、w_id をインクリメント。拡張語内の語境界。

3

POS と word_pos の対応

pos は各形態素の品詞タグ。word_pos は w_id グループ全体の代表品詞で、 グループ内の最初の非PX形態素の品詞（語彙的な品詞、語根類）が使われる。PX（接頭辞）は後続要素の品詞に吸収される。

Case A — 通常: 先頭 POS がそのまま word_pos になる

w_id グループの先頭形態素が PX でない場合、その POS がグループ全体の word_pos になる。

Case B — PX 接頭辞: 後続の非 PX 形態素の POS に揃える

先頭が NPX (名詞接頭辞) などの PX 系タグの場合、word_pos はそれをスキップして次の非PX形態素の品詞を採用する。

w_id = 2 「o-hanasi」

NPX - N → N - N
PX 系タグの例: NPX（名詞接頭辞）、VPX（動詞接頭辞）など。 末尾が PX で終わるタグはすべてスキップ対象。 グループ内に非PXが存在しない場合は、フォールバックとして先頭形態素のPOSがそのまま使われる。
